# 相册月历 (Photo Calendar) 产品需求文档 (PRD)

## 1. 项目背景
用户希望记录每一天的美好瞬间，通过照片的形式将一年的记忆串联起来。传统的日历以文字和日期为主，而本项目旨在开发一个“相册月历”网页，让用户每天上传一张喜欢的照片到对应的日期格子中，最终形成一个充满回忆的视觉化月历。

## 2. 产品目标
- 开发一个简单易用的网页应用。
- 实现以月为维度的日历视图。
- 允许用户点击日期格子上传或替换照片。
- 确保存储的照片在用户下次访问时依然可见（数据持久化）。
- 界面美观，以照片展示为核心。

## 3. 用户故事
- **作为一名用户**，我希望打开网页就能看到当前的月历视图，以便我查看日期待办或照片。
- **作为一名用户**，我希望能够切换月份（上个月/下个月），以便回顾过去或规划未来。
- **作为一名用户**，我希望点击某一个日期的格子，能够从我的设备中选择一张照片上传。
- **作为一名用户**，我希望上传的照片能够填满该日期的格子，且**不被日期数字遮挡**，以便完整欣赏照片。
- **作为一名用户**，我希望我的照片能够被自动保存，刷新页面后不会丢失。
- **作为一名用户**（可选），我希望能够将填满照片的月历导出为一张图片，分享给朋友。

## 4. 功能需求

### 4.1 日历核心功能
- **月视图展示**：
  - 显示当前年份和月份（例如：2026年 2月）。
  - 星期栏（周一至周日）。
  - 自动计算并排列当月的所有日期。
  - 支持显示上个月末尾和下个月开头的日期（通常置灰或留空，本项目建议留空或保持格子但不允许上传，聚焦当月）。

- **导航**：
  - “上个月”和“下个月”切换按钮。
  - “回到今天”按钮。

### 4.2 照片管理
- **上传照片**：
  - 点击日期格子触发文件选择器。
  - 支持常见图片格式（JPG, PNG, WEBP等）。
  - 图片上传后自动进行缩放/裁剪以适应格子大小（`object-fit: cover`）。
  
- **查看/替换**：
  - 鼠标悬停显示操作提示（如“更换照片”）。
  - 点击已上传照片的格子可重新上传替换。

- **数据存储**：
  - **MVP版本（最小可行性产品）**：使用浏览器本地存储 (IndexedDB 或 LocalStorage)。
    - *优点*：无需后端，开发快，隐私性好（数据在本地）。
    - *缺点*：更换设备或清空缓存后数据会丢失。
  - **进阶版本**（可选）：对接云端存储或后端数据库。

### 4.3 界面交互 (UI/UX)
- **布局**：
  - 顶部：年份、月份、导航按钮。
  - 中部：7列网格布局。
  - 单元格：正方形或接近正方形，保证照片展示效果。
- **样式**：
  - 极简风格，背景色主要为白色或浅灰。
  - **日期显示逻辑**：
    - **未上传照片时**：日期数字正常显示在格子内（例如左上角），方便用户识别。
    - **已上传照片时**：**日期数字自动隐藏**，以展示完整的照片内容。
    - **交互**：鼠标悬停在照片上时，日期数字浮现（或显示半透明遮罩层包含日期），提示当前日期。
  - 响应式设计：在电脑端和手机端都能良好展示。

## 5. 技术栈建议
- **前端框架**：React 或 Vue 3 (推荐使用 Vite 构建)。
- **样式库**：Tailwind CSS (快速构建网格和响应式布局)。
- **状态管理/存储**：
  - `dexie.js` (IndexedDB 的封装库，用于存储大量图片数据，LocalStorage 容量可能不足)。
  - 或者简单的 `File System Access API` (如果只做本地应用)。
- **图片处理**：`html2canvas` (如果需要导出功能)。

## 6. 开发计划
1. **项目初始化**：搭建 React/Vue + Tailwind 环境。
2. **日历逻辑实现**：生成日历网格数据。
3. **UI 组件开发**：头部导航、日历格子组件。
4. **图片上传与展示**：实现文件选择和预览。
5. **数据持久化**：集成 IndexedDB 存储图片数据。
6. **优化与测试**：样式微调，移动端适配测试。
